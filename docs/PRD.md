# PRD — Lumi

## 产品目标

Lumi 是一个桌面 AI 宠物。它常驻桌面角落，提供轻量聊天、记忆和状态反馈，让 AI 助手从“需要打开的工具”变成“持续存在的陪伴”。

核心体验优先级：

1. 陪伴感
2. 本地优先和隐私
3. 轻量、低打扰
4. 可扩展的角色表现

## 用户画像

- 长时间面对电脑工作的独立开发者、学生、创作者。
- 已经使用 AI 工具，但不想频繁切换浏览器或独立客户端。
- 喜欢桌宠、Live2D、像素/Sprite 角色表现。
- 关注本地数据和隐私。

## 解决的问题

- AI 对话入口割裂，需要切换应用。
- AI 工具缺乏持续存在感和角色状态。
- 普通桌宠缺少实用价值。
- 云端聊天产品对隐私和本地模型支持不足。

## 非目标

当前阶段不做：

- 云同步 / 账号系统。
- 多角色同时在线。
- 语音输入 / TTS。
- 移动端。
- 插件市场。
- AI 联网搜索。

## 关键流程

### 首次/日常使用

```text
启动应用
  -> 角色出现在桌面角落
  -> 点击聊天按钮
  -> 输入消息
  -> Rust 调用 LLM
  -> Lumi 回复
  -> 状态 thinking -> talking -> happy -> idle
```

### 设置和换模型

```text
右键菜单或托盘打开设置
  -> 扫描 public/models/
  -> 选择模型和类型
  -> 配置 provider / API Key / model
  -> 保存
  -> settings emit refresh-model
  -> main remount DesktopPet
```

## 优先级

### P0：稳定和基础体验

- 构建稳定。
- 渲染失败可回退。
- 设置保存后模型刷新可靠。
- 文档与代码同步。

### P1：陪伴感增强

- Sprite 分层资产完善。
- 更多低频动作和交互动作。
- 鼠标 hover / 双击 / 视线跟随。
- 窗口位置记忆。

### P2：AI 能力增强

- 多轮上下文管理。
- Markdown 回复。
- 流式输出。
- 记忆重要性和过期策略。
- 向量检索。

## 成功标准

- 主窗口轻量、稳定、低打扰。
- 用户无需打开浏览器即可完成基础 AI 对话。
- 模型加载失败不影响应用可用性。
- 本地数据路径清晰，用户隐私边界明确。
- 后续功能能在不重写核心架构的前提下演进。
