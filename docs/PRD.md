# PRD — Lumi

---

## 项目目标

让 AI 助手以"桌宠"形态常驻桌面。用户不需要打开浏览器或单独应用，随时与角色互动，角色记得你、有情绪、有状态。

---

## 用户画像

- 独立开发者 / 学生，长期对着屏幕工作
- 习惯 AI 工具，但不想每次都切换到浏览器
- 对动漫角色有亲近感，希望工具有温度
- 注重隐私（本地模型 / 本地存储）

---

## 核心问题

1. AI 对话入口太割裂（需要切换应用）
2. AI 没有"持续存在感"，每次对话像新人
3. 桌宠应用只有动画，没有实用价值
4. 现有桌宠不支持私有化 / 本地 AI

---

## MVP 范围

| 功能 | 说明 |
|---|---|
| 桌宠角色渲染 | Live2D 或 PNG Sprite，有状态动画 |
| 状态机 | 5 状态 + 时间驱动规则 |
| AI 对话 | 支持 4 个主流 provider |
| 记忆系统 | 自动摘要 + 注入上下文 |
| 设置 UI | 模型切换 + LLM 配置 |
| 截图隐藏 | Windows 平台 |
| 本地持久化 | SQLite + localStorage |

---

## 非目标（明确不做）

- 不做云端同步 / 账号系统
- 不做多角色同时在线
- 不做语音输入 / TTS
- 不做移动端
- 不做插件市场（当前阶段）
- 不做 AI 联网搜索（工具调用）

---

## 用户流程

```
安装启动
  → 角色出现在屏幕角落（emoji 降级 or Live2D/Sprite）
  → 点击聊天按钮 → 输入消息 → AI 回复
  → 角色进入 talking 状态 → 2s → happy → 6s → idle

设置流程
  → 系统托盘 → 打开设置
  → 选择模型（列表自动扫描 public/models/）
  → 配置 LLM provider + API Key
  → 保存 → 主窗口热重载模型

无操作
  → 45s 无操作 → sleepy
  → 22:00-07:00 → sleepy
  → 任意交互 → wakeUp → idle
```

---

## 核心页面

| 页面 | 入口 | 职责 |
|---|---|---|
| 主窗口 | 启动时 | 角色展示 + 聊天入口 + 快速设置 |
| 聊天面板 | 主窗口内滑出 | 消息收发 |
| 设置窗口 | 托盘菜单 | 模型选择 + AI 配置 |

---

## 功能优先级

| P0（核心） | P1（重要） | P2（增值） |
|---|---|---|
| 角色渲染 + 状态机 | 分层 Sprite 视差 | 流式输出 |
| AI 对话 | 记忆分级 | Markdown 渲染 |
| 记忆系统 | 窗口位置记忆 | 向量记忆检索 |
| 设置 UI | 开机自启 | 鼠标视线跟随 |
| 截图隐藏 | 全局快捷键 | 天气集成 |
